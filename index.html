SDUI Platform

Server-Driven UI platform for multi-brand mobile applications

Welcome to the documentation for the Server-Driven UI (SDUI) Platform — a hybrid architecture that powers multi-brand mobile applications from a single codebase.

What is the SDUI Platform?

The SDUI Platform enables a single React Native codebase to serve multiple branded mobile apps (iOS and Android) whose UI layouts, content, feature flags, and A/B tests are all controlled from a central server and CMS — without requiring app store submissions.

This platform was built as a Proof of Concept demonstrating how three mobile brands can share over 80% of their codebase while maintaining distinct visual identities and feature sets.

Key Capabilities

  • Server-Driven UI — Screen layouts defined as JSON schemas, rendered dynamically on the client
  • Multi-Brand Support — Three distinct brand themes from one codebase (Premium, Youth, Value)
  • Instant Updates — Change UI content via CMS without app store submissions
  • Feature Flags — Toggle features per brand with rollout percentage control
  • A/B Testing — Deterministic variant assignment for layout experiments
  • Personalization — Segment-based content (prepaid, postpaid, business)
  • Analytics — Built-in event tracking for impressions, clicks, and screen views
  • OTA Updates — Over-the-air code updates via Expo Updates
